PHP-START Comander - CLI Generator

Versión: 1.1
Módulo actualizado: Generador Dinámico de Módulos CRUD (`comander/modules.php`)

La utilidad CLI en comander/modules.php permite orquestar de forma automática toda la infraestructura necesaria para desarrollar rápidamente módulos o endpoints.

🚀 ¿Cómo se usa?

Abre la terminal de comandos de Laragon/Powershell y sitúate en la raíz del proyecto. El formato del comando es:

php comander/modules.php create <nombre_plural> [nombre_singular] [--context=admin|api|home|ajax]

Ejemplo Básico (Módulo "Products"):

php comander/modules.php create products product --context=admin
  • products: El identificador base (usado para la carpeta, el archivo de rutas base y el menú general).
  • product: Término singular (usado dinámicamente para subrutas lógicas como product/new). Si no se le proporciona, la herramienta intentará deducirlo cortando la terminación en 's' o 'es'.

📁 ¿Qué Estructura Genera?

Para la capa administrativa (--context=admin o --context=home), el script replicará el estándar CRUD agrupado imitando al módulo nativo de users:

  • 1. Acciones (Controllers): Crea internamente controladores aislados en actions/list.action.php, actions/new.action.php, actions/edit.action.php, y actions/delete.action.php listos para recibir la lógica de validación y base de datos.
  • 2. Vistas HTML: Genera esqueletos HTML en views/list.view.php, views/new.view.php, y views/edit.view.php, enlazados y renderizados bajo el layout general.
  • 3. Múltiple Routing (router.php): Despliega automáticamente un mapeador de rutas base, segmentadas de la siguiente manera: 
    Router::route('products')
Router::route('product/new')
Router::route('product/edit/{id}')
Router::route('product/delete/{id}')
    Cada ruta incluye la seguridad nativa a través de ->middleware('auth_admin') y las comprobaciones de permisos individuales (->permission('products.new')).
  • 4. Menú Sidebar Plegable (menu.php): Envuelve automáticamente las etiquetas de navegación de tu sistema en formato de menú desplegable. 
    $productsGroup = Sidebar::group('Products', 'circle');
$productsGroup->item('Nuevo Product', admin_route('product/new'))->can('products.new');
$productsGroup->item('Lista de Products', admin_route('products'))->can('products.list');
  • 5. Auto-Discovery & Auto-Registro: Tras crear los archivos pesados, la herramienta buscará y abrirá dinámicamente el archivo del registro maestro (app/admin/modules.php) y se activará a sí mismo concatenando el array general (ej. 'products' => true) garantizando que puedes arrancar la página al vuelo sin tocar dependencias.

⚡ Contextos Variables (APIs)

Puedes cambiar por completo el propósito del generador utilizando otra bandera. Por ejemplo:

php comander/modules.php create products --context=api

Ese comando omitirá las vistas e interfaces Sidebar que usa el motor, generando en su lugar un enrutador CRUD directo preparado para retornar respuestas o consumos mediante json_encode() limpio.

🔄 Reset Logos (reset-logos.php)

Utilízalo para restaurar rápidamente el favicon, el logo oscuro/claro y la metadata OpenGraph de la aplicación a sus valores por defecto, tanto en la base de datos como en la carpeta de subidas.

php comander/reset-logos.php

¿Qué ocurre tras bambalinas al ejecutarlo?

  • Forza valores JSON predeterminados en la tabla options (Claves: favicon, white_logo, dark_logo, og_image).
  • Purga recursívamente toda la caché de imágenes que esté guardada en storage/uploads/site.
  • Restituye imágenes frescas cargándolas desde la carpeta en crudo comander/images/site.

🔑 Sincronizador de Permisos (sync_permissions.php)

Esta utilidad permite mantener la tabla permissions de la base de datos sincronizada con las declaraciones ->permission() encontradas en todos los archivos router.php del proyecto.

php comander/sync_permissions.php

Características:

  • Fuente de Verdad: El código es el que manda. Si un permiso no está en los archivos, se elimina de la base de datos.
  • Auto-detección: Escanea automáticamente los contextos admin, front, api y ajax.
  • Integridad: Mantiene los IDs de permisos existentes para evitar conflictos con las asignaciones de roles.

🚀 Migrador de Base de Datos (migrate.php)

Utilidad para ejecutar archivos SQL de forma secuencial y automatizada. Ideal para instalaciones rápidas o actualizaciones de esquema.

php comander/migrate.php [fresh]

Comandos:

  • php comander/migrate.php: Ejecuta en orden alfabético/numérico todos los archivos .sql que comiencen con un número en la carpeta db/.
  • php comander/migrate.php fresh: Borra todas las tablas de la base de datos y vuelve a ejecutar los archivos de la carpeta db/. ⚠️ Usa esto solo en desarrollo.

📦 Respaldo de Base de Datos (backup.php)

Genera un volcado completo (esquema y datos) de la base de datos actual en un archivo SQL portable.

php comander/backup.php

Características:

  • Portabilidad: No requiere mysqldump instalado en el sistema, utiliza el motor nativo de PDO.
  • Ubicación: Guarda los archivos en db/backup_YYYY-MM-DD_HHMMSS.sql.
  • Aislamiento: Estos archivos son ignorados automáticamente por el migrador (ya que no empiezan por un número).

🧹 Limpiador de Caché (reset-cache.php)

Elimina todos los archivos temporales generados por el framework para forzar una recarga limpia de configuraciones y rutas.

php comander/reset-cache.php

¿Qué limpia?

  • Rutas: Borra la caché de rutas del sistema (storage/cache/router.cache.php), permitiendo que nuevos cambios en router.php se reflejen de inmediato si el sistema tiene activado el modo producción.
