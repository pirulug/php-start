ActionBtn

Descripción

La clase ActionBtn proporciona un constructor fluido (Fluent Builder) para la generación semántica de botones y enlaces en aplicaciones PHP.

Permite crear botones consistentes con estilos predefinidos (Bootstrap), integrar iconos de múltiples librerías (FontAwesome, Bootstrap Icons, Feather Icons), y manejar automáticamente la validación de permisos de usuario.

Además, cuenta con integración nativa para lanzar confirmaciones de SweetAlert en acciones destructivas y tiene lógica inteligente para comportarse como <a>, <button type="button"> o <button type="submit"> según el contexto.

Ejemplos de uso

1. Botón de Enlace Genérico (.btn-outline-secondary)

// Genera un enlace estándar
<?= ActionBtn::link('/admin/users')
  ->text('Volver')
  ->icon('fas fa-arrow-left'); ?>

2. Botón Guardar / Principal (.btn-primary)

// Inteligente: Al no pasar URL, asume que es para un formulario y genera un <button type="submit">
<?= ActionBtn::save() ?>

// Si se pasa URL, genera un enlace <a> con aspecto de botón primario
<?= ActionBtn::save('/admin/post/publish')->text('Publicar Ahora') ?>

3. Botón de Edición (.btn-success)

<?= ActionBtn::edit('/admin/users/edit/5') ?>

4. Botón de Eliminación con SweetAlert (.btn-danger)

// Genera un botón con atributos sa-* para ser interceptados por SweetAlert JS
<?= ActionBtn::delete('/admin/users/delete/5') ?>

5. Botón Ver Detalles (.btn-info)

<?= ActionBtn::view('/admin/users/profile/5') ?>

6. Botón Cancelar o Cerrar (.btn-secondary)

// Inteligente: Al no pasar URL, genera un <button type="button"> (Útil para cerrar Modales)
<?= ActionBtn::cancel()->attrs('data-bs-dismiss="modal"') ?>

// Si pasas URL, genera un enlace para retroceder
<?= ActionBtn::cancel('/admin/dashboard') ?>

7. Botones de Archivar y Exportar (.btn-warning / .btn-dark)

<?= ActionBtn::archive('/admin/posts/archive/12') ?>
<?= ActionBtn::export('/admin/reports/sales/csv') ?>

8. Validación de Permisos Dinámica

// El botón SOLO se renderizará (dibujará en el HTML) si el usuario tiene el permiso "users.delete"
<?= ActionBtn::delete('/admin/users/delete/5')
  ->can('users.delete') ?>

9. Uso de Iconos (FontAwesome, Bootstrap, Feather)

// FontAwesome o Bootstrap Icons (usa atributo class="")
<?= ActionBtn::link('/ruta')->icon('bi bi-person') ?>
<?= ActionBtn::link('/ruta')->icon('fas fa-user') ?>

// Feather Icons (Se auto-convierte a data-feather="")
<?= ActionBtn::link('/ruta')->icon('feather-user') ?>

10. Modificar Clases y Atributos HTML extra

// Sobrescribiendo clases y añadiendo atributos data-*
<?= ActionBtn::link('/ruta')
  ->text('Abrir Modal')
  ->classes('btn btn-lg btn-outline-primary text-uppercase')
  ->attrs('data-bs-toggle="modal" data-bs-target="#miModal"') ?>

11. Personalizar Mensajes de SweetAlert (Solo en ActionBtn::delete)

<?= ActionBtn::delete('/admin/products/delete/10')
  ->saTitle('¿Eliminar producto?')
  ->saText('Se borrará todo su inventario en las sucursales.') ?>

12. Forzar el Tipo de Elemento HTML

// Fuerza a que un enlace se convierta en un <button type="submit">
<?= ActionBtn::edit('')->text('Actualizar Cambios')->asSubmit() ?>

// Fuerza a que sea un <button type="button"> para usarlo con JS
<?= ActionBtn::save('')->text('Guardar por AJAX')->asButton()->attrs('id="btnAjax"') ?>

13. Flujo Completo Recomendado en Tablas

<td class="text-end">
  <?= ActionBtn::view('/admin/user/5')->can('users.read') ?>
  <?= ActionBtn::edit('/admin/user/edit/5')->can('users.update') ?>
  <?= ActionBtn::delete('/admin/user/delete/5')->can('users.delete') ?>
</td>
